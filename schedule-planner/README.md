Планировщик расписания

Интерактивное веб-приложение для управления расписанием и событиями, разработанное на чистом JavaScript (Vanilla JS) без использования фреймворков.

Описание проекта

**Планировщик расписания** — это функциональное веб-приложение, которое позволяет пользователям:
- Создавать, редактировать и удалять события
- Организовывать события по категориям
- Фильтровать события по категориям и поисковому запросу
- Сортировать события по дате, времени и названию
- Отмечать события как выполненные
- Сохранять все данные в браузере (localStorage)

Основные функции

### 1. Управление событиями
-  **Добавление события** — форма с полями: название, дата, время, категория, описание
-  **Редактирование события** — изменение данных через модальное окно
-  **Удаление события** — удаление с подтверждением
-  **Отметить как выполненное** — переключение статуса события

### 2. Фильтрация и поиск
-  **Поиск по названию и описанию** — реальный поиск в реальном времени
-  **Фильтрация по категориям** — выбор из 6 категорий
-  **Сортировка** — по дате (возрастание/убывание), времени, названию
-  **Сброс фильтров** — быстрое возвращение к полному списку

### 3. Категории событий
-  **Работа** — рабочие задачи и проекты
-  **Встреча** — встречи и совещания
-  **Личное** — личные дела и задачи
-  **Здоровье** — медицинские процедуры и занятия спортом
-  **Образование** — занятия и обучение
-  **Прочее** — другие события

### 4. Интерфейс
-  **Адаптивный дизайн** — работает на мобильных, планшетах и ПК
-  **Современный дизайн** — градиентные фоны и анимации
-  **Уведомления** — сообщения об ошибках и успешных операциях
-  **Валидация данных** — проверка всех введенных данных
-  **Автосохранение** — все данные сохраняются в localStorage

##  Установка и запуск

### Требования
- Современный веб-браузер (Chrome, Firefox, Safari, Edge)
- Локальный веб-сервер (опционально, но рекомендуется)

### Способ 1: Прямое открытие

1. Клонируйте репозиторий:
```bash
git clone https://github.com/alexkirjner/js.git
cd schedule-planner
```

2. Откройте файл `index.html` в браузере:
- Двойной клик на файл `index.html`
- Или перетащите файл в окно браузера

### Способ 2: С использованием локального сервера

#### Python 3
```bash
cd schedule-planner
python -m http.server 8000
```
Затем откройте в браузере: `http://localhost:8000`

#### Python 2
```bash
cd schedule-planner
python -m SimpleHTTPServer 8000
```

#### Node.js (http-server)
```bash
npm install -g http-server
cd schedule-planner
http-server
```

#### Live Server в VS Code
Установите расширение "Live Server" и нажмите "Go Live"

## 📖 Использование

### Добавление события
1. Заполните форму в разделе "Добавить новое событие"
2. Укажите название, дату, время и категорию (обязательные поля)
3. Добавьте описание (опционально)
4. Нажмите кнопку "Добавить событие"

### Фильтрация и поиск
- Выберите категорию в фильтре "Фильтр по категории"
- Введите текст в поле "Поиск" для поиска по названию или описанию
- Выберите способ сортировки из выпадающего меню
- Нажмите "Сбросить фильтры" для очистки всех фильтров

### Редактирование события
1. Нажмите кнопку "Изменить" на карточке события
2. Измените необходимые данные в модальном окне
3. Нажмите "Сохранить"

### Отметить как выполненное
- Нажмите кнопку "✓ Готово" для отметки события как выполненного
- Выполненное событие будет показано полупрозрачным с зачеркиванием названия

### Удаление события
1. Нажмите кнопку "Удалить" на карточке события
2. Подтвердите удаление в диалоговом окне

## Примеры использования

### Пример 1: Создание расписания рабочей недели
```
1. Добавить события:

2. Отфильтровать по категории "Работа" и "Встреча"

```

```
1. Добавить события:
   - "Прием у врача" — категория "Здоровье"
   - "Тренировка в спортзале" — категория "Здоровье"
   - "Медицинский осмотр" — категория "Здоровье"

2. Поиск по слову "спортзале"

```

```
1. Добавить события:
   - "Лекция по математике" — Вторник 10:00, категория "Образование"
   - "Практикум по программированию" — Четверг 14:00, категория "Образование"

2. Сортировать по времени
3. Отслеживать выполнение всех учебных мероприятий
```

##  Структура проекта

```
schedule-planner/
├── index.html          # Главная HTML-страница приложения
├── styles.css          # Стили CSS (адаптивный дизайн)
├── js/
│   ├── app.js         # Основной модуль приложения (координация)
│   ├── storage.js     # Модуль управления localStorage
│   ├── eventManager.js # Модуль обработки событий (фильтр, сортировка)

└── README.md          # Этот файл
```

### Описание модулей

| Модуль | Описание |
|--------|---------|
| **app.js** | Главный координатор всего приложения, управляет потоком данных между модулями |
| **storage.js** | Управление локальным хранилищем (localStorage), сохранение и загрузка событий |
| **eventManager.js** | Валидация, фильтрация, сортировка и форматирование событий |
| **ui.js** | Управление DOM, отображение событий, обработка пользовательского ввода |

##  Ключевые особенности кода

### 1. Модульная архитектура
Каждый функциональный компонент разделен в отдельный модуль с использованием IIFE (Immediately Invoked Function Expression) для изоляции области видимости.

### 2. Валидация данных
Все входные данные проверяются перед сохранением с помощью функции `validateEvent()`.

### 3. Защита от XSS
HTML-спецсимволы экранируются при отображении пользовательского ввода через функцию `escapeHtml()`.

### 4. Реактивное обновление интерфейса
При любых изменениях данные автоматически перезагружаются и интерфейс обновляется через `loadAndDisplayEvents()`.

##  Адаптивность

Приложение полностью адаптивно и корректно отображается на:
-  Мобильных телефонах (от 320px)
-  Планшетах (от 768px)
-  Десктопах (от 1024px)

Используется CSS Grid и Flexbox для гибкого макета, все элементы масштабируются автоматически.

## Безопасность

-  Все пользовательские данные экранируются перед выводом в HTML
-  Валидация на клиентской стороне предотвращает ввод некорректных данных
-  Защита от XSS-атак через экранирование HTML
-  Подтверждение перед удалением критичных данных
